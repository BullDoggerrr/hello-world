# hello-world


В качестве БД - PostgreSQL, параметры подключения прописывал в application.properties.

В ней три сущности: employees(сотрудники), employee_positions(должности), shifts(смены) - по каждой из них в соответствующих пакетах model, repository и controller лежат соответсвтующие классы/интерфейсы.
В exception - обработка ситуаций, когда не находятся в БД данные - выдаёт более удобоваримую информацию в JSON'е вместо пустого поля и т.п.

На данный момент в проекте всё кидается JSON'ами.

Для employee и employee_positions сущностей прописаны стандартные @GET, @POST, @PUT, @DELETE - запросы, тестил все из них пока только для employee_positions с помощью Postman API.

Для shifts те же запросы, но дополнительно написаны @GET-запросы поинтереснее: для вывода всей смены по дате и вывода смен для конкретного сотрудника, работают нормально, но есть сомнения касательно getShiftsByEmployeeId метода.
Дело в том, что в проекте у меня в Shift-сущности (model) лежит ссылка на Employee вместо его id, чтобы реализовать @ManyToOne-соответствие. Из-за этого приходится в ShiftController прописывать @Autowired EmployeeRepository, т.к. нужно каким-то образом по id выцепить ссылку на Employee.
В связи с этим вопрос - всё ли ок в этом случае со стороны логики или обращаться в контроллере (Shift) к чужому репозиторию (Employee) - не есть хорошо?
